用hexo建立自己的GitPages博客

Preparation

  • GitHub注册账号（本例中CoderMonkie）
  • Nodejs安装（本例中Win64环境）
  • 编辑工具（本文使用Visual Studio Code）

1. Create a repository on GitHub

在注册好的GitHub网站，点New，创建新的Repository。
填写Reposition名即网站名，格式为[你的用户名].github.io，如：
CoderMonkie.github.io

选择Public，初始化带readme文件，ignore可不选，版权信息可不选。



2. Install hexo

安装hexo

  • 自己找个路径新建一个文件夹（本例中CoderMonkeyBlog），在该路径下，用npm命令初始化

    这样，就创建出package.json文件，然后就可以安装hexokinase了

    通过查看版本看是否安装成功。

  • Edit [_config.yml] 修改配置文件
    主要修改以下Site，Url，Deployment，照实填写即可

3. Use hexo to create a new post

使用hexo发布文章

  • 创建一篇文章

由此在source/_posts文件夹下生成“hexoBlogGitPages.md”的markdown文件，
编辑此文件内容，就是你要发布的文章内容。语法自行百度markdown。

4. 先在本地查看生成的结果

用generate命令，生成文件

再用serve命令，启动localhost

这时候查看该 http://localhost:4000/
地址就能看到发布后的博客效果了。（没有问题的话下一步就是发布到GitPages上）

5. 部署到GitPages上

安装 hexo-deployer-git

如果配置没有改好的话，此时修改配置文件[_config.yml]

配置好之后，clean命令清除站点文件，deploy命令重新生成站点文件并推送到github指定的库上

clone时用的ssh，配置文件里相应的填了ssh的repo的url，
出以下问题：(没有访问权限，验证失败)

查了下没有解决，直接改成https然后重新clone的。
（Clone的不一样了，需要修改配置里的repo）

重新做了一遍然后执行[hexo deploy]，挣扎了几秒后就成功了。

6. View the site

查看博客发布后的结果
https://codermonkie.github.io/
